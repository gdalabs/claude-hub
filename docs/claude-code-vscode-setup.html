Claude Code とは？

Claude Code は Anthropic 公式の CLI（コマンドラインインターフェース）ツール です。 ターミナルから Claude に直接指示を出し、コードの生成・編集・デバッグ・リファクタリングなどをAIと対話しながら行えます。

あなた
(自然言語で指示)
→
Claude Code
(ターミナル)
→
コード編集
ファイル操作・実行

このガイドの流れ

2つのフェーズに分けて進めます：

PhaseDescription
Phase 1: ターミナル まず素のターミナルで Claude Code をインストール・認証・体験する。CLI の基本を掴む。
Phase 2: VS Code ターミナルで慣れたら VS Code に移行。Extension + 内蔵ターミナルでフル活用。
Why Terminal First? Claude Code の本体は CLI です。VS Code Extension も内部で CLI を呼んでいます。 ターミナルで基本を理解しておくと、VS Code での操作もスムーズに理解できます。
PHASE 1: Terminal
ターミナルで Claude Code を使えるようにする

Step 1: 事前準備

  • macOS / Linux / WSL — Windows はWSL経由で利用
  • Node.js 18+ — node -v で確認
  • Anthropic アカウント — Claude Code の認証に必要

Node.js のインストール（未インストールの場合）

# macOS (Homebrew) brew install node # または nvm を使用（推奨） curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash source ~/.zshrc # or ~/.bashrc nvm install --lts
確認: インストール後、node -v と npm -v が表示されれば OK。
Pro Plan 必須: Claude Code は Pro Plan ($20/mo) 以上で利用可能です。無料プランでは利用できません。Max Plan ($200/mo) だとレート制限が大幅に緩和されます。

Step 2: Claude Code のインストール

npm install -g @anthropic-ai/claude-code
Permission error? macOS で permission エラーが出る場合：
npm install -g @anthropic-ai/claude-code --cache /tmp/npm-cache

インストール確認

claude --version

Step 3: 初回認証

ターミナルを開いて、任意のディレクトリで claude を実行します。

1
ターミナルで claude を実行
macOS なら Terminal.app、iTerm2 など何でも OK。
2
ブラウザで認証
自動でブラウザが開きます。Anthropic アカウントでログイン。
3
認証完了 → ターミナルに戻る
ターミナルに戻ると Claude Code の対話モードが開始しています。
Terminal
$ claude

Claude Code v1.x.x

Opening browser for authentication...
Authenticated successfully!

? What can I help you with?
> （ここに指示を入力）

Step 4: ターミナルで使ってみる

プロジェクトに移動して起動

cd ~/projects/my-app claude

試してみよう

以下のような指示を入力してみてください：

Claude Code
# プロジェクトの理解
> このプロジェクトの構成を説明して

# ファイルを読む
> package.json の中身を見せて

# コードを書く
> Hello World を表示する index.html を作って

# バグ修正
> src/utils.ts のエラーハンドリングを修正して
Point: Claude Code はプロジェクト内のファイルを自動で読み取り、コンテキストを理解します。 「このファイルを修正して」のように自然言語で指示するだけで OK。

Permission（権限の確認）

Claude Code はファイル編集やコマンド実行の前に確認を求めます：

OptionDescription
Allow once今回だけ許可
Allow for sessionこのセッション中は許可
Allow always今後常に許可（設定に保存）
Deny拒否
Security: 最初は「Allow once」で様子を見て、信頼できる操作だけ「Allow always」にするのが安全です。

Step 6: CLAUDE.md を作成する

CLAUDE.md はプロジェクトのルートに置く設定ファイルです。 Claude Code がプロジェクトを理解するための「取扱説明書」として機能します。

自動生成（おすすめ）

Claude Code のセッション中に /init と入力すると、プロジェクトを自動解析して生成してくれます。

# Claude Code セッション内で /init

CLAUDE.md に書くこと

SectionExample
プロジェクト概要 何をするプロジェクトか、技術スタック
ビルド・実行コマンド npm run dev, npm run build, npm test
コーディング規約 言語、フォーマッタ、命名規則
注意点・制約 触ってはいけないファイル、特殊な設定

サンプル

# CLAUDE.md ## Project E-commerce web app built with Next.js + TypeScript. ## Commands - `npm run dev` — Start dev server (port 3000) - `npm run build` — Production build - `npm test` — Run Jest tests ## Conventions - Use TypeScript strict mode - Components in `src/components/`, pages in `src/app/` - Use Tailwind CSS for styling
Checkpoint: ここまでで Phase 1 は完了です。ターミナルから Claude Code を起動し、プロジェクトの理解・コード生成・ファイル編集ができるようになりました。 次は VS Code と統合して、より快適な開発環境を作ります。
PHASE 2: VS Code
エディタと統合してフル活用する

Step 7: VS Code の準備

  • VS Code — code.visualstudio.com からインストール
  • code コマンド — ターミナルから VS Code を開けるようにする

code コマンドの有効化

1
VS Code を開く
アプリケーションから VS Code を起動。
2
Command Palette を開く
Cmd + Shift + P（macOS）/ Ctrl + Shift + P（Linux）
3
"Shell Command: Install 'code' command in PATH" を実行
これでターミナルから code . が使えるようになります。

プロジェクトを VS Code で開く

cd ~/projects/my-app code .

Step 8: Claude Code Extension のインストール

1
拡張機能サイドバーを開く
Cmd + Shift + X（macOS）/ Ctrl + Shift + X（Linux）
2
"Claude Code" を検索
発行元が Anthropic であることを確認。
3
Install をクリック
インストール後、サイドバーに Claude アイコンが表示されます。
Alternative: コマンドラインからもインストールできます：
code --install-extension anthropic.claude-code
前提: Phase 1 で CLI の claude がインストール・認証済みであること。Extension は CLI をバックエンドとして使用します。

Extension でできること

FeatureDescription
サイドバー Chat VS Code 内で Claude と対話。コードのコンテキストが自動で渡される。
ファイル編集 Claude の提案する変更を diff 表示で確認 → Accept / Reject。
Terminal 連携 内蔵ターミナルでの claude コマンドとシームレスに連携。
ステータスバー Claude Code の接続状態・トークン使用量をリアルタイム表示。

Step 9: VS Code で快適に使うコツ

1. ターミナルを分割する

VS Code のターミナルを分割して、片方で Claude Code、もう片方で通常のコマンド（npm run dev など）を実行すると効率的です。

Shortcut: Cmd + \（macOS）でターミナルを分割。

2. 変更の確認は Source Control タブで

Claude Code がファイルを編集すると、VS Code の Source Control（Cmd + Shift + G）で差分を確認できます。 変更内容を目で確認してからコミットする習慣をつけましょう。

3. Auto Save を有効にする

Claude Code がファイルを書き換えた後、エディタが自動で最新状態を読み込みます。

// VS Code settings.json に追加 { "files.autoSave": "afterDelay", "files.autoSaveDelay": 1000 }

4. Git との組み合わせ

Claude Code で大きな変更をする前に、ブランチを切っておくと安心です：

git checkout -b feature/claude-experiment
Tip: Claude Code に直接「git commit して」と頼むこともできます。コミットメッセージも自動生成してくれます。

5. Extension と CLI の使い分け

Use CaseRecommended
ちょっとした質問・修正 Extension サイドバー — エディタから離れずに操作
大きな変更・複数ファイル操作 内蔵ターミナル CLI — より細かい制御が可能
プロジェクト外の作業 外部ターミナル CLI — ディレクトリを自由に移動

ProblemSolution

npm bin -g のパスを PATH に追加。
または npx @anthropic-ai/claude-code で直接実行。
認証エラー claude /logout → 再度 claude で認証し直す。
npm install -g で permission error
npm install -g @anthropic-ai/claude-code --cache /tmp/npm-cache
または nvm を使って Node.js を管理する（root 権限不要）。
VS Code Extension が動かない CLI の claude が先にインストール・認証済みであることを確認（Phase 1）。
Extension は CLI をバックエンドとして使用します。
レート制限に引っかかる /cost で使用量を確認。
/compact でコンテキストを圧縮するとトークン消費を抑えられる。
Max Plan への切り替えも検討。
ファイル変更がエディタに反映されない VS Code の Cmd + Shift + P → "Revert File" で最新を読み込み。
